# Latex Resume Builder

A desktop app that turns a simple form into a fully formatted PDF resume — no manual editing in Word, no fighting with alignment. Fill in your details through a Tkinter GUI, hit submit, and get a polished, LaTeX-generated resume.

## Quick Start

Clone the repo, then run one script — it installs everything (Python dependencies + LaTeX) and launches the app for you.

**Linux/macOS:**
```bash
git clone https://github.com/Janvi9090/resume-builder
cd resume-builder
bash setup.sh
```

**Windows:**
```
git clone https://github.com/Janvi9090/resume-builder
cd resume-builder
setup.bat
```
(or just double-click `setup.bat` in File Explorer)

The script checks for Python, Tkinter, and LaTeX, installs whatever's missing, then opens the app. On Windows, if LaTeX isn't found, it'll point you to the MiKTeX installer — just run that once, then re-run `setup.bat`.

## How it works

1. **Fill out the form** — a Tkinter desktop app with mandatory fields (name, email, contact) and optional, toggleable sections (Objective, Education, Skills, Projects, Experience, Achievements, Profiles/Social Links).
2. **Add as many entries as you need** — Education, Skills, Experience, and Projects all support adding multiple entries dynamically.
3. **Submit** — your data is saved to a structured JSON file (`data/resume_data.json`).
4. **Auto-generate** — a script reads that JSON, fills a LaTeX template using Jinja2, and compiles it into a PDF with `pdflatex`.
5. **Done** — your resume is saved to `output/resume.pdf`.

## Architecture

```
Tkinter GUI (User Input)
        ↓
User selects sections (checkbox-based)
        ↓
Dynamic form input collection
        ↓
User confirmation
        ↓
JSON file generation
        ↓
Auto-trigger Python generator script
        ↓
LaTeX template filled with JSON data
        ↓
PDF compiled
        ↓
Saved in /output folder
```

## Features

- Fully interactive Tkinter GUI, no web browser or internet required
- Optional sections toggled on/off with checkboxes
- Dynamically add multiple entries for Education, Skills, Experience, and Projects
- Multiple social/profile links (GitHub, LeetCode, LinkedIn, etc.) with clickable display text
- Choose a custom heading color for your resume — quick presets (Light Blue, Black, Dark Blue) or a full color picker
- Input validation (email format, 10-digit contact number)
- JSON-driven architecture — your data is stored in one clean, structured file
- Automatic LaTeX escaping, so special characters in your input never break the PDF
- Fully offline and lightweight

## Tech stack

- **Python 3** — core language
- **Tkinter** — desktop GUI
- **JSON** — data storage layer
- **Jinja2** — template rendering
- **LaTeX (pdflatex)** — PDF generation

## Manual Setup

If you'd rather not use the setup script, or want to understand what it's doing under the hood:

**1. Clone the repo**
```bash
git clone https://github.com/Janvi9090/resume-builder
cd resume-builder
```

**2. Create a virtual environment**
```bash
python3 -m venv venv
source venv/bin/activate      # On Windows: venv\Scripts\activate
```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Install LaTeX** (if not already installed)

On Debian/Ubuntu/Pop!_OS:
```bash
sudo apt install texlive-latex-base texlive-latex-extra texlive-fonts-recommended
```

On macOS:
```bash
brew install --cask mactex-no-gui
```

On Windows, install [MiKTeX](https://miktex.org/download).

**5. Run the app**
```bash
python3 gui.py
```

Fill in the form, click Submit, and check `output/resume.pdf`.

## Notes

- `output/` and `data/resume_data.json` are gitignored, since generated files and personal data shouldn't be committed.
- If PDF generation fails, check `output/resume.log` for the full LaTeX compilation log.
- `setup.sh` needs to be run with `bash setup.sh` (not `sh setup.sh`) since it uses bash-specific syntax.
